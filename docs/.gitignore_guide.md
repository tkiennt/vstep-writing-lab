# .gitignore Configuration Guide

This project uses comprehensive `.gitignore` files to exclude unnecessary files from version control.

## Root Level `.gitignore`

Located at: `/vstep-writing-lab/.gitignore`

### What's Ignored:

#### 1. **Node.js & Package Managers**
```
node_modules/          # npm dependencies
.npm/                  # npm cache
.eslintcache          # ESLint cache
```

#### 2. **Environment Files**
```
.env                   # Main environment (create from .env.example)
.env.local            # Local overrides
.env.*.local         # Environment-specific local files
```

#### 3. **Next.js Build Output**
```
.next/                # Next.js build directory
out/                  # Static export directory
build/                # Generic build folder
dist/                 # Distribution folder
```

#### 4. **Package Lock Files**
```
package-lock.json     # Use yarn.lock or pnpm-lock.yaml instead
yarn.lock            # Keep if using Yarn
pnpm-lock.yaml       # Keep if using pnpm
```

#### 5. **IDE & Editors**
```
.vscode/             # VS Code settings
.idea/               # IntelliJ IDEA settings
*.swp, *.swo         # Vim swap files
*~                   # Editor backup files
```

#### 6. **Qoder AI Cache** ⭐
```
.qoder/              # Qoder AI cache and configurations
```

#### 7. **Visual Studio (Backend)**
```
.vs/                 # Visual Studio workspace
bin/                 # Compiled binaries
obj/                 # Object files
*.user, *.suo        # Visual Studio user files
```

#### 8. **Testing & Coverage**
```
coverage/            # Test coverage reports
.nyc_output/         # NYC test coverage data
```

#### 9. **Firebase**
```
.firebase/           # Firebase local emulator data
.firebaserc          # Firebase project configuration
firebase-debug.log   # Firebase debug logs
```

#### 10. **OS Files**
```
.DS_Store           # macOS metadata
Thumbs.db           # Windows thumbnail cache
.Spotlight-V100     # macOS Spotlight index
.Trashes            # macOS trash info
```

#### 11. **TypeScript**
```
*.tsbuildinfo       # TypeScript build info
```

#### 12. **Error Logs**
```
build_errors.txt    # Custom build error logs
business_errors.txt # Custom business error logs
```

#### 13. **Temporary Files**
```
tmp/, temp/         # Temporary directories
*.tmp, *.temp       # Temporary files
```

## Setup Instructions

### After Cloning:

1. **Install Web Dependencies:**
   ```bash
   cd web
   npm install
   ```

2. **Setup Environment:**
   ```bash
   cp .env.example .env.local
   # Edit .env.local with your credentials
   ```

3. **Run Development:**
   ```bash
   npm run dev
   ```

### After Pulling Updates:

```bash
# Clean and reinstall
cd web
rm -rf node_modules .next
npm install
npm run dev
```

## Common Mistakes

### ❌ Wrong: Committing `.env.local`
```bash
git add .env.local  # DON'T DO THIS!
```

### ✅ Right: Using `.env.example`
```bash
git add .env.example  # Commit the template
```

### ❌ Wrong: Committing `node_modules`
```bash
git add node_modules  # NEVER!
```

### ✅ Right: Committing `package.json`
```bash
git add package.json  # YES!
```

## Checking What's Tracked

### See What Git Ignores:
```bash
git status --ignored
```

### Check If File Is Ignored:
```bash
git check-ignore -v path/to/file
```

### Force Add (If Really Needed):
```bash
git add -f filename  # Use with caution!
```

---

## Cleaning Up Previously Committed Files

If you accidentally committed files that should be ignored:

```bash
# Remove from git tracking but keep locally
git rm --cached -r node_modules/
git rm --cached -r .next/
git rm --cached .env.local

# Commit the removal
git commit -m "Remove files that should be in .gitignore"
```

---

## Best Practices

1. **Always use `.env.example`** as a template
2. **Never commit API keys or passwords**
3. **Keep lock files** (`yarn.lock`, `pnpm-lock.yaml`)
4. **Commit configuration files** (tsconfig, tailwind.config, etc.)
5. **Use `.gitignore` before creating files**, not after
